# Learn ai-news-briefing — chapter index

This is the long-form companion to the [main README](../../README.md). The README is the elevator pitch + reference; these chapters are the deep dive. Each one is self-contained (you can read them in any order), 600–1500 words, and ends with cross-references.

Each chapter has an accompanying **7-minute video overview** generated by Google NotebookLM. Watch the video first if you're a visual learner, then come back to the chapter for the file:line specifics.

## How to use this directory

- **Reading for understanding** — start with [00-overview](./00-overview.md), then [01-high-level-flow](./01-high-level-flow.md), then jump to whichever per-agent chapter sparks your interest.
- **Forking the project** — read [00-overview](./00-overview.md) → [03-trigger-and-runtime](./03-trigger-and-runtime.md) → [22-fork-guide](./22-fork-guide.md). Then dig into individual agents only as you swap them.
- **Presenting it** — chapters [01-high-level-flow](./01-high-level-flow.md), [02-architecture-layers](./02-architecture-layers.md), [17-distribution-aws](./17-distribution-aws.md), [21-tech-stack-and-tricks](./21-tech-stack-and-tricks.md) carry mind-map diagrams as well as videos.
- **Debugging a daily run** — [19-visibility-email](./19-visibility-email.md) tells you what each panel of the email means; [20-cost-and-fallbacks](./20-cost-and-fallbacks.md) tells you which fallbacks fire under load.
